Test Plan for “StudyBuddy App”

1. Introduction
The purpose of this test plan is to verify the functionality and usability of the “StudyBuddy” app — a mobile application designed to help students organize study sessions, set reminders, and track progress.

2. Objectives
- Ensure that all core app features work correctly.
- Identify and document any defects before release.
- Verify usability and performance across different devices.

3. Scope
Testing will include:
- User registration and login.
- Creating and editing study tasks.
- Setting reminders and notifications.
- Progress tracking dashboard.
- Logout functionality.

Out of scope: Integration with third-party cloud storage.

4. Test Strategy
- Testing Type: Manual testing.
- Test Levels: Unit, Integration, and System Testing.
- Environment: Android 13, iOS 17, Chrome browser (web version).
- Tools: JIRA for defect tracking, Google Sheets for test case documentation.
